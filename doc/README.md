# 文档目录说明

本目录用于存放项目研发过程中的文档记录：

- STATUS_YYYY-MM-DD.md：阶段性状态快照（时间、构建、思路、风险与下一步）。
- CHANGELOG.md：关键变更与里程碑。
- BUILD_LOG.md：构建/训练/运行中出现的问题与解决方案（按时间追加）。
- ERROR_LOG.md：错误堆栈摘录与分析（需要时创建）。

撰写原则：
1. 面向重现与交接，避免只写结果不写原因。
2. 重要决策要写“为何这样做”与退路方案。
3. 遇到数据问题，记录原始路径与示例文件名，保护隐私（不要放真实病人信息）。
